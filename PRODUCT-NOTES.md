# 产品定位与公开发行说明

当前版本：V0.2.1。

## 一句话定位

微信年轮：把一年里容易被忽略的微信小事，变成一份只属于自己的年度档案。

## 最适合传播的结果

- “今年我发了 12,680 条微信，撤回了 43 条。”
- “最常出现的时间是晚上 11 点。”
- “发出 52 个红包，但分享图不展示金额。”
- “这一年有 284 个活跃日，最长连续聊天 31 天。”
- “我的表达方式：文字 62%，表情包 14%，图片 11%。”

## 后续值得增加的统计

- 第一句和最后一句消息的日期（默认不展示内容）
- 最长语音、累计语音分钟数、视频通话次数与时长
- 深夜消息、节假日消息、周末/工作日对比
- 最活跃单日、最安静月份、连续活跃纪录
- 发过多少图片、文件、链接、位置、名片、小程序和拍一拍
- 群聊/私聊比例、主动发起会话次数、平均回复间隔（需可靠方向字段）
- 联系人关系热度变化（只做趋势，不给“感情好坏”结论）
- 朋友圈图文/视频比例、评论与点赞趋势
- 红包平均金额、最大单笔转账、收支方向变化（默认隐私模式）
- 与上一年对比、群聊年度报告、双人共同年报

## 不应该承诺的能力

- 找回已经彻底删除且无任何残留的聊天记录
- 100% 统计所有撤回记录
- 仅凭电脑缓存保证朋友圈完整
- 仅凭聊天卡片精确还原红包/转账最终金额
- 在用户不知情的情况下读取他人账号或设备

## 传播增长

分享卡片必须做到“有趣但不泄密”：默认只显示汇总数量，不显示联系人、原文和金额。卡片底部附产品名即可，不强制水印，不要求上传数据，不以“分享解锁”绑架用户。可信度标签会成为与同类产品最明显的差异点。

## 成本判断

PWA 首发可以接近 0 元。静态站可使用免费托管，分析在浏览器本机完成。公开 GitHub 仓库可用 Actions 自动打包 Mac Apple 芯片、Mac Intel 与 Windows。未签名安装包会有系统提示；苹果和 Windows 代码签名并非小几十元能够解决，因此应推迟到用户量已经证明产品值得投入时。
