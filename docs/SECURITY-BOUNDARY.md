# 安全边界与适配器规则

## V0.2.0 明确做到

- 只读取用户主动选择的 CSV/JSON/TXT。
- Electron 开启上下文隔离、沙箱、Web 安全与导航限制。
- 不保存原始文件，不建立服务器数据库，不发送遥测。
- HTTPS 外链只交给系统浏览器，不在应用中执行第三方网页。

## 主程序明确不做

- 扫描微信进程、注入 DLL、读取进程内存
- 获取或保存 DbKey、支付密码、登录凭据
- 解密原始 WCDB、绕过系统完整性保护
- 分析他人账号、后台监控、远程取证

## 未来独立适配器的准入条件

1. 只处理当前操作系统登录用户本人有权访问的数据。
2. 首次执行前清楚展示读取位置、输出位置和风险，必须主动确认。
3. 不联网传输密钥或数据库；日志自动脱敏。
4. 输出微信年轮标准 JSON/CSV，主程序通过文件边界消费。
5. 可单独卸载，不影响主程序导入官方账单和已有文件。
6. 不要求关闭 macOS SIP；不偷偷重签或替换微信客户端。
7. 针对微信版本维护兼容矩阵，失败时明确停止，不猜测密钥或数据。
8. 独立进行许可证与当地法律审查，再决定是否发布二进制。

这条边界让项目既能继续研究可用性，又不会为了“一键读取”牺牲用户设备安全和整个开源项目的可持续性。
