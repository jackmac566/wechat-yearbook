# 安全边界与适配器规则

## V0.2.1 明确做到

- 只读取用户主动选择的 CSV/JSON/TXT。
- Electron 开启上下文隔离、沙箱、Web 安全与导航限制，使用隔离的 `app://` 协议而不是给予任意 `file://` 页面额外权限。
- 拒绝网页嵌入、未知外链、开发者工具入口和相机/麦克风/蓝牙等所有系统权限请求。
- 生产包关闭 `ELECTRON_RUN_AS_NODE`、`NODE_OPTIONS`、命令行调试参数，启用 ASAR 完整性校验并只从 ASAR 加载应用代码。
- macOS 包不声明未使用的相机、麦克风、蓝牙权限，也不允许任意网络传输。
- 不保存原始文件，不建立服务器数据库，不发送遥测。
- HTTPS 外链只交给系统浏览器，不在应用中执行第三方网页。

## 发行包验证

- GitHub Actions 在打包前运行锁定依赖安装、完整测试、版本元数据检查与 `npm audit`。
- 构建任务默认只有源码只读权限；仅发布任务获得 Release、OIDC 和证明写入权限。
- 第三方 GitHub Actions 固定到完整提交 SHA。
- 每个 Release 包含 `SHA256SUMS.txt`，并为安装包生成 GitHub Artifact Attestation 构建来源证明。
- 工作流先把全部附件放入草稿再发布，且发现同名 Release 时直接失败，不覆盖已发布附件，兼容 GitHub Immutable Releases。

以上措施可以显著降低供应链和桌面外壳风险，但不能把“未付费签名”变成“已签名”。零成本包在 macOS/Windows 上仍可能显示未知开发者或未知发布者；下载者必须核对仓库、校验和与构建证明。

## 主程序明确不做

- 扫描微信进程、注入 DLL、读取进程内存
- 获取或保存 DbKey、支付密码、登录凭据
- 解密原始 WCDB、绕过系统完整性保护
- 分析他人账号、后台监控、远程取证

## 未来独立适配器的准入条件

1. 只处理当前操作系统登录用户本人有权访问的数据。
2. 首次执行前清楚展示读取位置、输出位置和风险，必须主动确认。
3. 不联网传输密钥或数据库；日志自动脱敏。
4. 输出微信年轮标准 JSON/CSV，主程序通过文件边界消费。
5. 可单独卸载，不影响主程序导入官方账单和已有文件。
6. 不要求关闭 macOS SIP；不偷偷重签或替换微信客户端。
7. 针对微信版本维护兼容矩阵，失败时明确停止，不猜测密钥或数据。
8. 独立进行许可证与当地法律审查，再决定是否发布二进制。

这条边界让项目既能继续研究可用性，又不会为了“一键读取”牺牲用户设备安全和整个开源项目的可持续性。
