# 微信年轮 V0.2.1

这是一次准确性、安全性和“全量已有文件导入”修复版。

## 重点修复

- 聊天记录、支付账单、朋友圈与微信年轮标准 JSON 可以一起多选，桌面端也可直接选择整个导出文件夹。
- 修复标准 JSON 重新导入后红包与转账统计归零的问题。
- 修复带时间戳的会话容器导致其内部消息被漏读的问题。
- 不再把缺少 `isOwn` 的朋友圈互动默认算作本人操作。
- 每个文件都会显示识别条数和跳过条数，异常行不再静默丢失。
- 支付账单只有在声明的日期范围覆盖完整年份时才标记为“完整”；多个连续账单可合并判断。
- 标准 JSON 可明确选择“仅当前年份”或“全部年份”，避免文件名写某一年却导出全部历史。

## 桌面安全

- Electron 改用隔离的 `app://` 协议，拒绝任意本地文件导航、网页嵌入与系统权限请求。
- 生产包关闭开发者工具、Node 环境变量和调试参数入口，启用 ASAR 完整性校验并只从 ASAR 加载应用代码。
- macOS 安装包移除相机、麦克风、蓝牙等未使用权限说明，并关闭任意网络加载声明。
- GitHub Release 增加 SHA-256 校验文件和构建来源证明；发布流程不再覆盖已有附件。
- 依赖审计为 0 个已知漏洞（以本次发布构建时的 `npm audit` 结果为准）。

## 仍然存在的边界

微信没有在 Mac 上提供“把全部聊天和朋友圈导出为 JSON/CSV”的官方按钮。本应用只分析你已经取得并主动选择的文件，不会关闭 SIP、提取微信密钥或扫描他人账号。彻底删除且没有残留的记录仍无法可靠检测。

macOS 和 Windows 安装包目前未购买商业代码签名证书，因此系统仍可能显示“未知开发者/未知发布者”。请只从本仓库 Release 下载，并使用随包提供的 `SHA256SUMS.txt` 核对文件。
