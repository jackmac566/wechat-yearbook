# 更新日志

V0.2.0 起的版本可由 GitHub Release、标签和 Actions 构建记录验证。V0.1.x 是项目迁入 GitHub 前保留下来的产品原型记录，不宣称存在对应 Git 标签或公开安装包。

## V0.2.1 — 2026-07-22

全记录导入、准确性与桌面安全修复版。

### 导入全部已有记录

- 聊天 JSON/CSV、微信支付账单、朋友圈 JSON/CSV 和微信年轮标准 JSON 可以一次多选并合并去重。
- 桌面端增加“选择整个导出文件夹”，自动忽略图片、视频、数据库等非结构化数据文件。
- JSON/CSV 解析移入本地后台线程；超过 50 MB 会先提示资源占用，不会上传。
- 数据页逐个显示文件名、成功识别条数、跳过条数和原因警告。

### 统计准确性

- 修复标准 JSON 重新导入后，`red_packet` / `transfer` 被归类为其他而导致金额与笔数归零。
- 修复会话容器自身带时间戳时，其内部消息被跳过的问题。
- 缺少 `isOwn` 的朋友圈记录不再默认归属于本人，并明确提示未计入数量。
- 官方账单只有在声明的起止范围覆盖完整自然年时才显示“完整”；多个相邻账单范围可以合并判断。
- 标准 JSON 明确区分“仅当前年份”和“全部年份”，不再出现文件名写某年却包含其他年份记录。
- 系统通知不再计入收到的普通消息；无发送记录时不再虚构 00:00 或 1 月为高峰。

### 桌面与发布安全

- Electron 使用隔离的 `app://` 协议，拒绝任意本地文件导航、网页嵌入和相机/麦克风等系统权限请求。
- 生产包关闭开发者工具、Node 环境变量和调试参数入口，启用 ASAR 完整性校验并只允许从 ASAR 加载应用代码。
- macOS 包移除相机、麦克风、蓝牙等未使用权限声明，并关闭任意网络加载。
- 移除存在安全公告的开发依赖链；发布时 `npm audit` 为 0 个已知漏洞。
- GitHub Actions 权限按任务拆分，第三方 Action 固定到提交；Release 附带 SHA-256 校验和与构建来源证明，且不再覆盖已有附件。

## V0.2.0 — 2026-07-22

双平台开源桌面预览版。

### 年报体验

- 默认进入一屏一个结论的沉浸式年报故事，降低原仪表盘的信息密度。
- 保留详细总览、聊天、关系、朋友圈与账单页面。
- 未导入的数据明确显示“未导入”，不拿零值冒充真实统计。

### 兼容与准确性

- 适配多种大小写字段、EchoTrace/WeFlow 风格字段和嵌套会话 JSON。
- 新增带 `schemaVersion` 的微信年轮标准 JSON 导入与导出。
- 修正本地时区边界下活跃日与连续活跃天数的计算。
- 新增兼容字段与标准格式往返测试。

### 开源与桌面发行

- 增加 Mac Apple 芯片、Mac Intel 与 Windows 的 GitHub Actions 自动打包。
- 增加 MIT License、第三方参考、隐私、安全、贡献、架构、数据格式和新手发布文档。
- 固定 npm 依赖版本并强化 Electron 的导航、沙箱与网页安全边界。
- 主程序不包含微信进程扫描、密钥提取和数据库解密；未来只能由可移除的独立适配器接入。

## V0.1.2 — 2026-07-21

真实数据与上传向导版。

### 真实

- 首页默认改为空白数据状态，未导入就不显示统计数字。
- 演示数据只在用户主动点击后出现，并持续标明为模拟记录。
- 清除导入后回到空白状态，不再自动恢复演示数据。

### 向导

- 增加微信支付个人对账 CSV 的手机申请、解压与上传步骤。
- 增加 Windows 与 macOS 聊天记录迁移、开源导出和文件选择说明。
- 增加朋友圈可获取范围、字段格式与不完整性提示。

### 适配与边界

- 适配 `wechat-export-macos` 的 `chat.json` 与 `chat.csv` 字段，正确识别“我”发出的消息、消息类型与撤回记录。
- 明确区分官方账单、第三方本地导出和实验性朋友圈数据。
- 明确提示 macOS 重签微信的安全影响；不建议普通用户直接操作，不建议关闭 SIP。
- 彻底删除且无残留的数据不会被恢复、估算或伪造。
